Event Delegation

Its a technique involving adding event listeners to a parent element instead of adding them to the descendant elements. The listener will fire whenever the event is triggered on the descendant elements due to even bubbling up the DOM. The benefits of this technique are:

  • Memory footprint goes down because only one single handler is needed on the parent element, rather than having to attach event handlers on each descendant.
  • There is no need to unbind the handler from elements that are removed and to bind the event for new elements.

Example:

We have a list below that has the id "parent-list" and it has several child elements:

  • Item 1
  • Item 2
  • Item 3
  • Item 4
  • Item 5

We want something to happen when each child element is clicked. You could add event listeners to each li element but when there are many li elements being added and removed, it could be bothersome.

Solution:

You add an event listener to the parent ul element and check the event object's target property to gain a reference to the actual clicked node.

            
                const myList = document.getElementById('parent-list')

                myList.addEventListener('click', function(e) {
                    const target = e.target;
                    if (target.matches('li')) {
                        target.style.backgroundColor = 'blue';
                    }
                })
